# JTR-136

- Date: 2026-03-02
- Scope: add loading/empty/error states for tickets list `all` view

## What changed

- Added explicit UI states for `/tickets` `all` tab:
  - loading: 7-column skeleton rows while RxDB subscription initializes,
  - empty (no tickets): `JEmptyState` with `+ New Ticket` CTA,
  - empty (filters active): `JEmptyState` with `Clear filters` action,
  - error: `JEmptyState` with retry action when RxDB query/subscription fails.
- Added page-level loading skeleton for `/tickets`:
  - unified skeleton for filters/tabs/content during auth/location bootstrap and first local data hydration,
  - keeps existing tab-specific (`all`) loading/empty/error states after initial hydration.
- Added subscription lifecycle state in page logic:
  - `ticketsLoading`,
  - `ticketsLoadError`,
  - retry and filter-reset handlers.
- Kept existing table/pagination path for normal loaded state.
