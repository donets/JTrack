# JTR-113

- Date: 2026-03-02
- Scope: implement ticket checklist card UI with persisted checklist state

## What changed

- Added dedicated component `TicketChecklistCard` for ticket detail sidebar checklist:
  - completion counter and progress bar,
  - checkbox list rendering from checklist items.
- Updated ticket detail page to use `TicketChecklistCard` instead of hardcoded inline checklist markup.
- Wired checklist toggles to persistent storage:
  - local checklist state mirrors ticket document checklist,
  - toggles call `repository.saveTicket({ checklist })`,
  - triggers `syncStore.syncNow()` after update.
- Added fallback default checklist template for tickets that still have empty checklist arrays.
